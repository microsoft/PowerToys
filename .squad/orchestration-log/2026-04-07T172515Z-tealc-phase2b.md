## Technical Details

**Resilient Loading Pattern:**
- Initial start timeout: 10s (`ExtensionStartTimeout`)
- Initial command load timeout: 10s (`CommandLoadTimeout`)
- Background start timeout: 60s (`BackgroundStartTimeout`)
- Background command load timeout: 60s (`BackgroundCommandLoadTimeout`)
- Extensions timing out on initial start retry via `StartExtensionWhenReadyAsync`
- Command loads timing out defer via `AppendCommandsWhenReadyAsync`

**Inner Types:**
- `ExtensionStartResult` — private nested class with `[MemberNotNullWhen]` attributes
- `CommandLoadResult` — private nested class with `[MemberNotNullWhen]` attributes
- `TopLevelObjectSets` — holds separated commands and dock bands
- `IsExtensionResult` — private nested record struct (avoids collision with same-named type in Models/ExtensionService.cs)

**Event Model:**
- `OnCommandsAdded`/`OnCommandsRemoved` fire with merged commands + dock bands
- Consumers responsible for splitting via `CombineTopLevelObjectSets` helper
- Fires `OnCommandProviderRemoved` on uninstall

**Package Catalog Integration:**
- Discovers WinRT extensions from package catalog
- Synchronous discovery under `_catalogLock` (blocking `Task.Run().Result` call for `IsValidCmdPalExtensionAsync`)
- Hot-install/uninstall handlers update tracking lists and fire appropriate events

**Concurrency:**
- Uses C# 13 `Lock` for synchronous sections
- Uses `SemaphoreSlim` for async-safe gating
- Package catalog events handled synchronously, then async work via `Task.Run`

**Constructor Dependencies:**
- `IPackageCatalogReference` — WinRT package catalog
- `ILoggerFactory` — creates typed loggers for self and `CommandProviderWrapper` instances
- `ISettingsService` — for settings access
