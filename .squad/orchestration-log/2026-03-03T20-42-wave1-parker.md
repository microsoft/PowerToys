## Task 2: JSON-RPC Connection Layer

**Deliverables:**
- `src/modules/cmdpal/Microsoft.CmdPal.UI.ViewModels/Services/JsonRpc/JsonRpcMessage.cs`
- `src/modules/cmdpal/Microsoft.CmdPal.UI.ViewModels/Services/JsonRpc/JsonRpcConnection.cs`

### JsonRpcMessage.cs

Implemented 4 message types:
- `JsonRpcRequest`: method + params + id
- `JsonRpcResponse`: result/error + id
- `JsonRpcNotification`: method + params (no id)
- `JsonRpcError`: standard JSON-RPC codes (-32700 to -32603)

### JsonRpcConnection.cs

Robust bidirectional communication manager:

**Transport:**
- LSP-style length-prefixed framing ("Content-Length: N\r\n\r\n" + JSON)
- UTF-8 encoding over process stdout/stdin

**Thread Safety:**
- `SemaphoreSlim` for write synchronization
- `Lock` for thread-safe ID generation
- `ConcurrentDictionary` for request correlation and notification handlers

**Request Management:**
- Auto-incrementing request IDs
- 10-second default timeout per request
- `TaskCompletionSource<JsonRpcResponse>` for each pending request
- Proper cancellation propagation via `CancellationToken`

**Background Read Loop:**
- Dedicated thread reads from stdout
- Byte-by-byte header parsing until `\r\n\r\n` delimiter
- Reads exact content length, handles partial reads gracefully

**Message Dispatch:**
- Responses: matched by ID to pending requests
- Notifications: dispatched to registered handlers by method name

**Error & Cleanup:**
- `OnError` event for read/parse failures
- `OnDisconnected` event for process exit
- Graceful disposal: cancels pending requests, waits for read loop, cleans up resources
